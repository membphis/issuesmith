Enter explore mode to think through ideas before creating an Issue.

Use the issuesmith-explore skill to:
- Open an exploratory conversation with no required structure
- Investigate the codebase and read project docs (`docs/`)
- Check existing GitHub Issues to avoid duplicates
- Compare options, sketch architecture, surface risks
- Transition to `/ism:create` when ready to formalize

**This is a thinking partner, not an implementation tool.** No code is written and no Issues are created.
