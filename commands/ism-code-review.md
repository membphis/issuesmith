Review code changes against requirements and code quality standards before merging or proceeding to the next task.

Use the issuesmith-code-review skill to:
- Diff the current branch against origin/main to determine the change scope
- Check five dimensions: Issue alignment, code quality, architecture, testing, production readiness
- Categorize issues by actual severity (Critical / Important / Minor)
- Output structured review with file:line references and clear verdict
- Act on results: fix Critical and Important issues before continuing

**Review early, review often. Catch issues before they cascade.**
